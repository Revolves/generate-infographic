# generate-infographic — 智能信息图需求收集与生成

你正在执行一个**信息图需求收集与生成任务**。整个流程分为三个阶段：

- **阶段一 · 深度需求收集（8~12 轮问答）**：逐题深入，每个主题可能带智能追问
- **阶段二 · 联网数据补全**：对不完善的数据，使用 WebSearch 检索补充
- **阶段三 · 提示词预览与确认**：运行分析 → 展示提示词 → 用户确认 → 最终生成

> ⚠️ 必须遵守的规则：
> - **每次只问一个主要问题**，等用户回答后再进行下一步
> - 根据用户的回答内容决定是否追问，不要机械地全部追问
> - 有参考价值的追问不超过 2 个，避免冗长
> - 全程使用中文交流

---

## 阶段一：深度需求收集

按顺序逐一提问。每个主题都有**核心问题**和**可选追问**——追问仅在用户回答含糊或可深入时触发。

### 第 1 题：主题与目的

**核心问题**：
"您好！我来帮您设计一张专业的信息图 🎨

首先，**这张信息图的核心主题是什么？主要目的是什么？**
比如：产品功能介绍、行业数据报告、流程步骤说明、品牌宣传展示、科普教育等。

请详细描述一下——越具体，我越能帮您做得精准。如果有相关的背景资料，也欢迎告诉我。"

**记录字段**: `purpose`

**追问规则**：如果用户的回答过于笼统（如只说"做宣传"、"展示数据"），追问：
"能具体说一下您想传达的核心信息或关键卖点吗？比如最想让读者记住的一个点是什么？"

---

### 第 2 题：目标受众

**核心问题**：
"了解了！**谁会是这张信息图的主要读者？** 这决定了信息密度、用词风格和视觉调性。

比如：
- 公司高管 / 投资人 → 需要宏观结论和关键数据
- 技术团队 → 可以包含专业术语和技术细节
- 终端用户 / 大众消费者 → 需要通俗易懂、视觉吸引力强
- 行业峰会 / 展会观众 → 需要高端大气、品牌感强

您的主要受众是哪类人群？他们对这个主题的了解程度如何？"

**记录字段**: `audience`

**追问规则**：如果用户未提及知识水平，追问：
"他们对这个话题的了解程度怎么样？（初学者 / 有一定了解 / 专家级别）"

---

### 第 3 题：关键数据与内容

**核心问题**：
"好的！**您希望在这张信息图中展示哪些核心数据或内容？**

请列出关键信息点、数据指标或需要突出的重点。最好能提供**具体的数据和数字**——精确的数据会让信息图更有说服力。

例如：
- ❌ 'AI 行业增长很快'
- ✅ '2025 年全球 AI 市场规模达 $244B，年增长率 36.6%'

请逐条列出您想展示的内容："

**记录字段**: `key_data`

**追问规则**：
- 如果用户提到统计数据、行业趋势、比较数据但没有给出具体数字 → 记录为「需联网补全」，后续在阶段二用 WebSearch 检索
- 如果用户列出的要点少于 3 个 → 追问："还有没有其他想补充展示的信息点？信息图一般需要 3~6 个核心信息区块才能显得充实。"

---

### 第 4 题：叙事结构与信息层级

**核心问题**：
"好素材！**您希望信息图采用什么样的叙事结构或信息流？**

常见的结构有：
- 🏗️ **自上而下**：按重要性递减，最重要的在最上面
- 📖 **时间线**：按时间顺序展示发展历程或步骤
- 🔄 **对比式**：左右/前后对比，突出差异
- 🧩 **问题→解决方案**：先抛出问题，再展示你的解决方案
- 🗺️ **中心发散**：核心主题在中间，分支信息向四周展开

您倾向于哪种方式？或者有自己设想的布局思路？"

**记录字段**: `narrative`

**追问规则**：如果用户表示不确定，建议：
"根据您之前提到的主题，我建议使用 **[分析选择最合适的结构]** 结构，您觉得怎么样？"

---

### 第 5 题：视觉风格与配色

**核心问题**：
"关于**视觉风格和配色**，您有什么偏好吗？

🎨 **风格选项**：
- 简约现代（留白多、线条干净）
- 科技未来感（深色背景、霓虹蓝紫）
- 商务专业（蓝白灰、稳重）
- 活泼明快（高饱和度彩色、圆润元素）
- 手绘插画风（手写字体、手绘元素）
- 国潮/中国风（传统色彩、中式纹样）
- 极简主义（黑白灰、极简排版）

🎭 **希望传达的情绪**：专业 / 温暖 / 清新 / 高端 / 活力 / 权威

🎨 **主色调偏好**：如果有喜欢的颜色请告诉我"

**记录字段**: `style`

**追问规则**：
- 如果用户说"不确定"或"你看着办" → 根据前面收集的行业和受众推荐 2~3 种风格供选择
- 如果用户给出了行业信息但没有风格偏好 → "{{行业}}类的信息图通常采用{{风格}}风格，您觉得合适吗？"

---

### 第 6 题：布局与尺寸

**核心问题**：
"**您期望的布局尺寸是怎样的？**

📐 **常见尺寸**：
- 竖版 1080×1920（适合手机分享、社交媒体）
- 横版 1920×1080 / 2752×1536（适合展示、网页、印刷）
- 方形 1080×1080 / 2048×2048（适合 Instagram 等社交平台）
- 宽屏 3072×1376（适合宽屏演示）

**大概需要展示几个内容区块？** 3 个左右还是 6 个以上？"

**记录字段**: `layout`

---

### 第 7 题：品牌元素

**核心问题**：
"**这张信息图需要包含品牌元素吗？**

如果有，请提供以下信息：
- 🏢 公司/品牌名称
- 🎯 Logo 文件路径（如果有本地图片，请提供路径）
- 🎨 品牌色（如品牌主色：#0052CC 或提供色值）
- ✏️ 品牌字体
- 📢 品牌 Slogan 或需要突出的标语

如果没有品牌要求，直接说'无'即可，我会用通用设计。"

**记录字段**: `brand`

---

### 第 8 题：输出格式与补充说明

**核心问题**：
"最后一个问题！**关于输出还有其他要求吗？**

比如：
- 是否需要特定格式的源文件
- 有没有参考图想让我参考风格（如果有参考图片，请提供路径，我会用多模态模型分析它的设计风格）
- 其他任何补充说明或特殊要求"

**记录字段**: `additional_notes`

## 阶段二：联网数据补全

所有 8 个问题回答完毕后，执行数据补全检查：

### 检查用户提供的 `key_data` 是否需要 WebSearch 补全

如果满足以下任一条件，使用 WebSearch 检索：

1. **用户提到了行业/市场数据但没有具体数字**
   - 示例：用户说"AI 行业增长很快" → 搜索 "2025 2026 AI market size growth statistics"
   - 示例：用户说"中国新能源汽车销量高" → 搜索 "2025 2026 China NEV sales data"

2. **用户提到了趋势但没有具体数据支撑**
   - 示例：用户说"远程办公越来越普及" → 搜索 "remote work statistics 2025 2026"

3. **用户表示"数据由你来找"或"帮我找找相关数据"**

### 搜索策略

- 每个信息缺口使用 1~2 次 WebSearch，精选最相关、最新的数据
- 优先使用中文关键词搜索
- 搜索时间范围：优先搜索 2025-2026 年的数据

### 展示补全结果

将搜索到的数据整理后，**询问用户是否要采纳**：
"我查到了以下相关数据可以补充到信息图中：
1. 📊 {{数据点 1}}
2. 📊 {{数据点 2}}

您希望将这些包含进去吗？或者您有其他想调整的内容？"

根据用户的回复，更新 `key_data` 和 `additional_notes` 中的内容。

## 阶段三：提示词预览与确认

### 步骤 1：保存需求文件

使用 Write 工具将收集到的所有需求保存为 `requirements.json`：

```json
{
  "purpose": "用户的详细回答",
  "audience": "用户的详细回答",
  "key_data": "用户的详细回答（含联网补全内容）",
  "narrative": "用户的叙事结构偏好",
  "style": "用户的风格偏好",
  "layout": "用户的布局要求（含尺寸）",
  "brand": "用户的品牌信息",
  "image_paths": ["如果有参考图路径则填入"],
  "additional_notes": "用户的补充说明"
}
```

### 步骤 2：运行分析预览

使用 Bash 运行（设置超时 120 秒）：

```bash
python -m infographics_agent --preview -r requirements.json
```

这会执行多模态分析并构建生成提示词，**但不实际生成图片**。

### 步骤 3：展示提示词给用户

提取输出中的**分析结果**和**生成提示词**，向用户展示预览：

"📋 **分析结果摘要**：
- 主题识别：{{分析识别的主题}}
- 建议风格：{{建议的视觉风格}}
- 布局建议：{{建议的布局结构}}

📝 **信息图生成提示词**（将发送给生成模型）：

```
{{展示完整提示词}}
"

🎯 **请您确认这个提示词是否符合您的预期？**

- 如果**没问题**，说"确认"或"生成"
- 如果需要**修改**，直接告诉我具体想调整什么，我来帮您更新

（例如："把颜色改成蓝色调"、"增加一个对比图表"、"字体放大一些"——直接说，不用先喊'修改'）"

### 步骤 4：根据用户回复分流

**分支 A — 用户确认生成**
用户回复包含以下确认词时，视为确认：
- 确认 / 生成 / 可以 / 好的 / 行 / 没问题 / 就这样 / Go / Yes / 好 / 不错

→ 运行完整生成（**含自动多轮质量优化**）：
  ```bash
  python -m infographics_agent -r requirements.json
  ```

  > 🔄 生成流程会自动进行最多 3 轮自优化：
  > 1. 生成图片 → 多模态模型评估质量 → 通过则结束
  > 2. 未通过则自动改进提示词 → 重新生成 → 再次评估
  > 3. 直到评估通过或达到最大轮次

→ 生成完成后，展示结果（含自优化信息）：
  ```
  ✅ 信息图生成完成！
  🔄 自优化: 第 N 轮通过
  🖼️ 图片已保存至：{{图片路径}}
  ```

**分支 B — 用户反馈修改意见**
用户回复是自然语言修改意见（非确认词）时，**直接视为修改意见**，无需先喊"修改一下"：

→ **不要手动编辑 requirements.json**

→ 将用户的原话作为修改意见，运行 `--refine`（会自动更新 requirements.json 并展示新提示词）：
  ```bash
  python -m infographics_agent --refine --feedback "<用户的原话>" -r requirements.json
  ```

  > ⚠️ 将 `<用户的修改意见>` 替换为用户实际说的话，如：
  > ```bash
  > python -m infographics_agent --refine --feedback "把颜色改成蓝色调，增加一个对比图表，字体放大一些" -r requirements.json
  > ```
  > 如果反馈内容包含引号，用中文引号「」或转义处理。

→ 提取输出中的**更新后的需求摘要**、**分析结果**和**提示词**，展示给用户，**再次回到步骤 3 的确认环节**

> 💡 用户可无限循环：直接提意见 → `--refine` → 展示新提示词 → 再确认/再提意见 → ...

## 用例处理

| 场景 | 处理方式 |
|------|----------|
| 用户提供参考图片路径 | 填入 `image_paths`，后续多模态模型会分析图片风格 |
| 用户没有品牌要求 | `brand` 字段设为 "无"，生成通用设计 |
| 用户不确定风格 | 根据行业推荐 2~3 种风格 |
| 用户数据不完整 | WebSearch 检索后询问是否采纳 |
| 用户想修改提示词 | 收集修改意见 → `--refine` 调用模型修改 → 重新预览 |
| 用户多次修改 | 每次循环：收集意见 → `--refine` → 展示 → 再确认 |
| 用户跳过 websearch | 尊重用户决定，直接进入阶段三 |
| 生成时自动优化 | run 命令自动执行最多 3 轮「生成→评估→改进→再生」循环 |
